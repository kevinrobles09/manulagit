INTRODUCCIÓN

Git es una herramienta práctica que permite a los desarrolladores de software controlar y distribuir las versiones y estados de sus códigos o proyectos de sistemas informáticos en los que se estén trabajando, de una manera fácil y eficiente.

Git se ha convertido en una herramienta indispensable a la hora programar, ya que permite la distribución de los códigos a las diferentes plataformas en las que estén vinculados los usuarios del proyecto, permitiendo así seguridad y permanencia de los datos en las estancias incluidas, como son: el equipo local, los equipos de los asociados y el servidor web. Además, su fácil vinculación con la red social de código Open Sourse conocida como GitHub se pueden compartir los repositorios creados, para su identificación, modificación y uso del mismo por la comunidad.

HISTORIA DE GIT Y GITHUB

En 2005, la relación entre la comunidad que desarrollaba el núcleo de Linux y la compañía que desarrollaba BitKeeper se vino abajo, y la herramienta dejó de ser ofrecida gratuitamente. Esto impulsó a la comunidad de desarrollo de Linux (y en particular a Linus Torvalds, el creador de Linux) a desarrollar su propia herramienta basada en algunas de las lecciones que aprendieron durante el uso de BitKeeper. Algunos de los objetivos del nuevo sistema fueron los siguientes:

  • Velocidad
  • Diseño sencillo
  • Fuerte apoyo al desarrollo no lineal (miles de ramas paralelas)
  • Completamente distribuido
  • Capaz de manejar grandes proyectos (como el núcleo de Linux) de manera eficiente (velocidad y tamaño de los datos)

Le genial plataforma GitHub fue creada por Chris Wanstrath , PJ Hyett , Tom Preston-Werner y Scott Chacon ayá por Febrero de 2008 en San Francisco – EEUU y hasta la fecha sigue estando ubicada en el mismo lugar

En Febrero de 2009 los integrantes de la compañía anuncian en una entrevista para Yahoo que GitHub ya había alcanzado más de 46 mil repositorios públicos, una buena cantidad que indicaba la popularidad que se iba a venir para Github. En Julio de ese mismo año 2009, GitHub alcanza los más de 135 mil repositorios.

GitHun es una plataforma en la Nube (Internet) que sirve como almacenamiento de código fuente y archivos de un proyecto de Software, Web, Aplicación Móvil, etc. todo esto de manera Gratuita, pero GitHub cuenta con su versión de Pago para poder usar otras funciones en su plataforma.

COMO INSTALAR GIT

COMANDOS TERMINAL

Los comandos en la terminal bash, permiten la interacción con la raíz de los sistemas de archivos, permitiéndonos de forma directa la manipulación de ficheros y directorios. A continuación algunos comandos importantes.

  • 1. crear carpeta: mkdir "nombre de carpeta" (sin comillas)
  • 2. borrar carpetas: rm -rf "nombre de la carpeta" (sin comillas)
  • 3. crear archivos: touch "nombre del archivo" (sin comillas)
  • 4. borrar archivos: rm "nombre del archivo" (sin comillas)
  • 5. mover en carpeta: cd cd ..
  • 6. ver elementos: ls
  • 7. limpiar terminal: clear 

COMANDOS GIT

También hay comandos en la terminal bash que funcionan o están diseñados para la manipulación y distribución de las versiones de los sistemas y proyectos que se estén desarrollando. A continuación algunos comandos indispensables.

CONFIGURACIÓN INICIAL DE GIT:

  • 1. git config --global user.email “correo electronico” (sin comillas) Establece el correo electrónico.
  • 2. git config --global user.name "nombre de usuario" Establece el nombre de usuario.
  • 3. git config --global color.ui true Establece el color.

SEGUNDA FASE: PREPARAR ARCHIVOS Y CREACIÓN DE REPOSITORIOS (STAGE):

  • 1. crear repo: git init "nombre de carpeta" (sin comillas)
  • 2. borrar repo: rm -rf .git
  • 3. volver repo a una carpeta: get init
  • 4. Estado de git: git status
  • 5. mover a stage: git add "nombre de archivo" (sin comillas)
  • 6. quitar de stage: git rm --cached "nombre del archivo" (sin comillas)
  • 7. agregar todo al stage: git add –A
  • 8. borrar archivos de stage y de carpeta: git rm -f "nombre del archvo" (sin comillas)
  • 9. saber si existe en directorio: git add -n "nombre del archivo" (sin comillas)

TERCERA FASE: SUBIR REPO A UN SERVER:

confirmar cambios: git commit -m "inicializar nuestro landing"
